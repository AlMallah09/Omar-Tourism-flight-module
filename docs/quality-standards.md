# Quality Standards

## Code Quality

- Use clear file and folder structure.
- Use meaningful function and variable names.
- Separate routes, services, schemas, models, and database logic.
- Avoid writing all logic in one file.

## API Quality

- Use RESTful endpoint naming.
- Return proper HTTP status codes.
- Return JSON responses.
- Validate request data.
- Document endpoints using FastAPI automatic documentation.

## Security Quality

- Do not store plain-text passwords.
- Do not hardcode database credentials.
- Use environment variables.
- Protect sensitive routes.
- Apply role-based access control.

## Database Quality

- Use primary keys.
- Use foreign keys.
- Use constraints.
- Prevent invalid seat availability.
- Keep relationships clear.

## Git Quality

- Commit changes regularly.
- Use clear commit messages.
- Do not push virtual environments.
- Do not push secret files.
