# Use Cases

## Purpose

This document defines the main use cases for the flight management and booking backend.

The system will be built using Python FastAPI and will initially use mock/sample data.

## Customer Use Cases

1. Register an account.
2. Log in to the system.
3. Search available flights.
4. View flight details.
5. Create a booking.
6. View personal bookings.
7. Cancel a booking.

## Administrator Use Cases

1. Log in as an administrator.
2. Add flight records.
3. Update flight records.
4. Delete flight records.
5. Manage seat availability.
6. View customer bookings.
7. Review basic operational data.

## Future Provider Integration Use Cases

1. Search flights using third-party provider APIs.
2. Retrieve real-time pricing.
3. Retrieve real-time availability.
4. Confirm bookings through provider APIs.
5. Store provider booking references locally.
