# Development Log – Phase 3

## Objective

Implement the booking management system and establish relationships between users, bookings, and flights.

## Tasks Completed

* Created Booking model and database table.
* Implemented booking schemas.
* Developed booking service layer.
* Created booking API routes.
* Established User ↔ Booking relationship.
* Established Flight ↔ Booking relationship.
* Implemented booking creation functionality.
* Added automatic total price calculation.
* Added seat availability validation.
* Implemented automatic seat deduction after successful booking.
* Added booking cancellation functionality.
* Implemented automatic seat restoration after cancellation.
* Tested all endpoints using Swagger UI.

## Result

The system now supports complete booking management. Users can create bookings, seat availability is validated automatically, pricing is calculated dynamically, and cancelled bookings restore flight seat availability correctly.

## Status

Completed Successfully.
